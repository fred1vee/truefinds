# TrueFinds AI Development Rules

## General Rules

This project is a long-term commercial project.

The code must always be production-ready.

Never generate demo code.

Never generate placeholder architecture.

Never simplify the UI.

Always think about scalability.

---

## Architecture

Before editing:

1. Read PROJECT_CONTEXT.md
2. Read TRUEFINDS_PRD.md (when available)
3. Read the current code.

Only then make changes.

---

## Design Rules

The design should feel:

- Premium
- Minimal
- Clean
- Modern
- Fast

Avoid unnecessary decorations.

Animations must feel smooth.

---

## Coding Rules

Always use:

- TypeScript
- Functional Components
- Reusable Components

Never duplicate code.

Keep files small.

Split logic into components.

---

## Component Rules

Every UI element should become a reusable component if it can be reused.

Examples:

Button

Card

Input

Modal

Badge

Navbar

Sidebar

Product Card

Tool Card

Statistic Card

---

## Styling Rules

Use Tailwind CSS.

Do not use inline styles.

Keep spacing consistent.

Prefer rounded corners.

---

## Performance

Always optimize:

Images

Rendering

Bundle size

Animations

---

## Git Rules

Never modify git configuration.

Never remove project files.

Never rename folders unless requested.

---

## Final Rule

Before writing code, think first.

Never rush.

Quality is more important than speed.
